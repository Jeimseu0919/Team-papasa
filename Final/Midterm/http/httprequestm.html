Request Methods

"Eight Standard Request Methods"

◾GET - a request to the server to retrieve the resource identified.

◾HEAD - similar to get, but the server must not return a message-body in response

◾POST - sent to submit html form data to the server. Content is not cached. (A notification to resubmit form data is often seen when a form is filled and then the page is refreshed or reloaded.)

◾PUT - uploads representation of URL

◾DELETE - delete the listed or specified resource or item

◾OPTION - returns supported HTTP methods by the server

◾TRACE - for debugging. Similar to ping for IP, sends a looping message from sender to target source.

◾CONNECT - converts requested connection to a TCP/IP tunnel


"Method Properties"

  • Safe methods - open and read only
    • GET, HEAD, OPTIONS, TRACE
  • Idempotent methods - Expected effect will be the same as that of a request
    • GET, HEAD, OPTIONS, TRACE, DELETE, PUT
  • Cacheable methods - A method with results that can be stored and reused later
    • GET, HEAD, POST

HTML is a format that tells a computer how to display a web page. The documents themselves are plain text files with special "tags" or codes that a web browser uses to interpret and display information on your computer screen. ◾ HTML stands for Hyper Text Markup Language ◾ An HTML file is a text file containing small markup tags ◾ The markup tags tell the Web browser how to display the page ◾ An HTML file must have an htm or html file extension

There are several different version of HTML ◾ Pre-standardization:     ● HTML Tags (http://info.cern.ch/hypertext/WWW/MarkUp/Tags.html), by Tim Berners-Lee, late 1991.     ● Hypertext Markup Language (HTML) Internet Draft, by Tim Berners-Lee and Dan Connolly, published by the IETF in mid-1993     ● HTML+ (https://www.w3.org/MarkUp/HTMLPlus/htmlplus_1.html) (Hypertext Markup Format) Internet Draft, by Dave Raggett, late 1993
◾Standards:     ● HTML 2.0 (RFC 1866 (https://tools.ietf.org/html/rfc1866), Nov 1995)         ● supplemental RFCs:     ● HTML 2.0 (RFC 1866 (https://tools.ietf.org/html/rfc1866), Nov 1995)             ○ Form-Based File Upload (RFC 1867 (https://tools.ietf.org/html/rfc1867), Nov 1995)             ○ HTML Tables (RFC 1942 (https://tools.ietf.org/html/rfc1942), May 1996)             ○ Client-Side Image Maps (RFC 1980 (https://tools.ietf.org/html/rfc1980), Aug 1996)             ○ Internationalization (i18n) of HTML (RFC 2070 (https://tools.ietf.org/html/rfc2070), Jan 1997)     ● HTML 3.2 (https://www.w3.org/TR/REC-html32) (W3C Recommendation, Jan 1997)     ● HTML 4.0 (https://www.w3.org/TR/REC-html40-971218/) (W3C Recommendation, Dec 1997)     ● HTML 4.01 (https://www.w3.org/TR/html4) (W3C Recommendation,Dec 1999)             ○ introduced deprecation of presentational elements and attributes in favor of style sheets             ○ Strict, Transitional, Frameset DTDs
